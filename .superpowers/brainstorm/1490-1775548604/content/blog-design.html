Building Mahjong

Over the past few months I've been working on building a multiplayer Mahjong game. This post walks through the architecture, the challenges of real-time game state, and how I approached the UI.

Architecture

The stack is straightforward: a React frontend communicating over WebSockets to a Node.js backend. Game state lives entirely on the server.

// Connect to game server
const socket = io(SERVER_URL);
socket.on('gameState', handleUpdate);

Each player connects to a room, and the server broadcasts state changes to all participants...
